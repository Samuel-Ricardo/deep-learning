1 Introdução
1.1 Sobre Este Laboratório
Neste laboratório, serão apresentados os conceitos fundamentais da estrutura de desen-
volvimento de Inteligência ArtificialMindSpore, com foco em aplicações práticas de Deep
Learning. O objetivo é fornecer uma introdução ao ambiente MindSpore, explorando sua

Inicialmente, serão abordados conceitos introdutórios relacionados ao framework

ender diferentes estratégias de construção, treinamento e avaliação de modelos de Deep
Learning para visão computacional.
Ao longo do laboratório, também serão investigadas etapas importantes do pipeline
de aprendizado de máquina, incluindo preparação dos dados, treinamento, inferência e
avaliação de desempenho dos modelos.

2.2 Procedimento
2.2.1 Introdução aos tensores
Tensor é uma estrutura básica de dados em computação de rede doMindSpore. Para
detalhes sobre tipos de dados em tensores, consulte a descrição do dtype no site oficial do
MindSpore.
Tensores de diferentes dimensões representam dados diferentes. Por exemplo, um ten-
sor 0-dimensional representa um escalar, um tensor 1-dimensional representa um vetor,
um tensor 2-dimensional representa uma matriz e um tensor 3-dimensional pode repre-
sentar os três canais de imagens RGB.
Os tensores doMindSporesão compatíveis com diferentes tipos de dados, incluindo
int8,int16,int32,int64,uint8,uint16,uint32,uint64,float16,float32,float64
ebool_, que correspondem aos tipos de dados deNumPy.
NoprocessocomputacionaldoMindSpore, otipodedadosintemPythonéconvertido
para o tipoint64definido e o tipo de dadosfloaté convertido para o tipofloat32
definido.
2.2.2 Criação de tensores
Durante a criação do tensor os tipos a seguir podem ser inseridos:float,int,Boolean,
tuple,list, eNumPy.array.Tuplee alistpodem armazenar apenas dados dos tipos
float,inteBoolean. O tipo de dados pode ser especificado durante a inicialização do
tensor. Se o tipo de dados não for especificado, os valores iniciais int, float e bool gerarão
tensores 0-dimensionais com os tipos de dadosmindspore.int32,mindspore.float32
emindspore.bool_, respectivamente. Os tipos de dados dos tensores 1-dimensionais
gerados pelos valores iniciaistupleelistcorrespondem aos tipos de dados de tensores
armazenados na tupla e lista. Se vários tipos de dados estiverem contidos, o tipo de
dados doMindSporecorrespondente ao tipo de dados com a prioridade mais alta será
selecionado (Boolean<int<float). Se o valor inicial éTensor, o tipo de dados é
tensor. Se o valor inicial éNumPy.array, o tipo de dados do tensor gerado corresponde
aNumPy.array.

def create_dataset(data_path, batch_size=18, training=True):
"""Define the dataset."""
data_set = ds.ImageFolderDataset(data_path, num_parallel_workers=8,
shuffle=True, class_indexing={’daisy’: 0, ’dandelion’: 1,
’roses’: 2, ’sunflowers’: 3, ’tulips’: 4})
# Realize o aprimoramento das imagens no conjunto de dados.
image_size = 224
mean = [0.485 * 255, 0.456 * 255, 0.406 * 255]
std = [0.229 * 255, 0.224 * 255, 0.225 * 255]

trans = [
CV.RandomCropDecodeResize(image_size, scale=(0.08, 1.0), ratio=
(0.75, 1.333)), CV.RandomHorizontalFlip(prob=0.5),
CV.Normalize(mean=mean, std=std), CV.HWC2CHW()
]

CV.Decode(),
CV.Resize(256),
CV.CenterCrop(image_size),
CV.HWC2CHW()
]
# Execute as operações de mapeamento de dados, em lote e de repetição.
data_set = data_set.map(operations=trans, input_columns="image",
num_parallel_workers=8)

Etapa 3– Criar o modelo MobileNetV2.
Os conjuntos de dados são fundamentais para o treinamento. Um bom conjunto de
dados pode melhorar significativamente a precisão e eficiência do treinamento.
A MobileNet é uma rede neural convolucional leve proposta pelo Google em 2017,
voltada para dispositivos móveis, embarcados e IoT. Comparada às CNNs tradicionais, a
MobileNet reduz significativamente o número de parâmetros mantendo boa precisão.
Além disso, os coeficientes de larguraαe resoluçãoβsão introduzidos para atender
diferentes cenários de aplicação.
Como a função de ativação ReLU pode causar perda de informações em representações
de baixa dimensão, a MobileNetV2 utiliza:
•Blocos residuais invertidos;
•Gargalos lineares.
Essa estratégia melhora a precisão e reduz ainda mais o tamanho do modelo.

Na estrutura residual invertida:
•Convolução1×1é utilizada para aumento de dimensão;
•Convolução profunda3×3é aplicada;
•Outra convolução1×1realiza redução de dimensão.
Essa estrutura é inversa à utilizada em blocos residuais tradicionais.
